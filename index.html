Qual é o resultado da query?

SELECT COUNT(*) FROM cliente c LEFT JOIN compra cp ON c.ID_CLIENTE = cp.ID_CLIENTE;

O LEFT JOIN preserva clientes sem compra e pode multiplicar linhas quando há mais de uma compra relacionada.

Trilhas

Seis caminhos para evoluir com ordem e prática.

As trilhas combinam fundamentos, SQL, estatística, indicadores e projetos para formar repertório aplicável.

01

Fundamentos de Dados

Tipos de dados, tabelas, qualidade da informação e indicadores básicos.

02

SQL Essencial

SELECT, WHERE, ORDER BY, GROUP BY, COUNT, SUM e AVG para análises do dia a dia.

03

SQL Intermediário

JOIN, LEFT JOIN, cardinalidade, COUNT DISTINCT, subqueries e CTEs.

04

Estatística para Dados

Média, mediana, moda, desvio padrão, distribuição e correlação.

05

Indicadores e KPIs

Metas, gaps, variação percentual, histórico, ranking e leitura de contexto.

06

Projetos Práticos

Análise de vendas, NPS, dashboard operacional e insights acionáveis.
